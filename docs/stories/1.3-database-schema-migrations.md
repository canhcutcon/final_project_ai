# Story 1.3: Database Schema & Migrations

## Status

Draft

## Story

**As a** developer,
**I want** the database schema created via Alembic migrations,
**so that** schema changes are version-controlled and reproducible.

## Acceptance Criteria

1. Alembic configured với MySQL dialect (`mysql+pymysql://`)
2. 5 bảng chính được tạo: `users`, `datasets`, `analysis_results`, `reports`, `pipeline_runs`
3. Bảng `datasets`: id (CHAR(36) UUID), filename, file_path, file_size, data_type, row_count, column_count, columns_info (JSON), uploaded_at, user_id (FK)
4. Bảng `analysis_results`: id, dataset_id (FK), model_used, config (JSON), total_anomalies, anomaly_ratio, scores (JSON), metrics (JSON), duration_seconds, created_at
5. Bảng `reports`: id, analysis_id (FK), language, content (TEXT), pdf_path, created_at
6. Bảng `pipeline_runs`: id, dataset_id (FK), status, current_step, config (JSON), result_id (FK), report_id (FK), started_at, completed_at, error_message
7. `alembic upgrade head` chạy thành công, `alembic downgrade -1` rollback đúng
8. Indexes trên các foreign key columns

  - [ ] Index trên `datasets.user_id`

### Patterns & Conventions

- **ORM Pattern**: SQLAlchemy 2.x declarative models with type hints
- **UUID as PK**: `CHAR(36)` in MySQL — generate with `str(uuid.uuid4())` in Python
- **JSON columns**: MySQL `JSON` type — used for flexible data (`columns_info`, `scores`, `metrics`, `config`)
- **Relationships**: SQLAlchemy `relationship()` with `backref` for reverse access
- **Naming**: Table names lowercase plural (`users`, `datasets`). Model names PascalCase singular (`User`, `Dataset`)
- **Nullable**: FK columns that are set later (e.g., `result_id`, `report_id` in `pipeline_runs`) must be `nullable=True`

### Tech Stack

- **SQLAlchemy**: 2.x — declarative models, async engine
- **Alembic**: latest — auto-generate migrations from model metadata
- **PyMySQL**: MySQL driver for SQLAlchemy (`mysql+pymysql://`)
- **MySQL**: 8.0+ — JSON column support, InnoDB engine (default)

### Notes from Previous Stories

- **Story 1.1**: `config.py` đã tạo với `DATABASE_URL` env var. Models folder structure đã scaffold.
- **Story 1.2**: MySQL service đã running via Docker Compose tại port `3306`.

### Testing

- **Framework**: pytest
- **Test location**: `{backend}/tests/test_models.py`
- **Test types**: Unit tests cho model creation, Integration tests cho migration
- **Minimum tests**:
  - Test User model creation và query
  - Test Dataset model với relationship to User
  - Test `alembic upgrade head` / `alembic downgrade -1` cycle
  - Test JSON column serialization/deserialization
