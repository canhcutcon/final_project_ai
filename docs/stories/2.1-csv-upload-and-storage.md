# Story 2.1: CSV Upload & Storage

## Status

Draft

## Story

**As a** user,
**I want** to upload a CSV file through the API,
**so that** my data is stored securely for analysis.

## Acceptance Criteria

1: `POST /api/v1/upload` nhận file CSV qua `multipart/form-data`
2: Validation: reject file > 100MB, reject file không phải `.csv` extension
3: File lưu vào MinIO bucket `datasets` với key `{user_id}/{dataset_id}/{filename}`
4: Metadata lưu vào bảng `datasets`: filename, file_path, file_size, row_count, column_count, columns_info (JSON chứa name, dtype, null_count, unique_count per column)
5: Response trả về `dataset_id` (UUID) và metadata summary
6: Trả `413 Payload Too Large` nếu file quá lớn
7: Trả `422 Unprocessable Entity` nếu CSV parse thất bại

## Dev Notes

### Relevant Source Tree

- `{backend}/app/api/v1/endpoints/upload.py` — route handler
- `{backend}/app/api/v1/router.py` — register upload routes
- `{backend}/app/api/deps.py` — `get_current_user`, `get_db` dependencies
- `{backend}/app/schemas/upload.py` — Pydantic DTOs
- `{backend}/app/models/dataset.py` — SQLAlchemy model
- `{backend}/app/services/data_service.py` — business logic
- `{backend}/app/services/storage.py` — MinIO client wrapper
- `{backend}/app/core/exceptions.py` — custom exceptions

### Patterns & Conventions

- Layered architecture: Endpoint → Service → Model
- All service methods `async def`
- Pydantic schemas for request/response validation (auto-enforced by FastAPI)
- Custom exceptions in `core/exceptions.py`, propagated via FastAPI exception handlers
- File naming: `snake_case.py`
- Import order: stdlib → third-party → local

### Tech Stack

- FastAPI (REST API, `UploadFile` for multipart)
- SQLAlchemy 2.x (async ORM, JSONB for `columns_info`)
- Alembic (migration)
- MinIO (S3-compatible, port 9000/9001)
- Pandas (CSV parsing, column profiling)
- PostgreSQL 15+ (JSONB support)
- JWT authentication via `{backend}/app/core/security.py`

### Testing

- Location: `{backend}/tests/`
- Framework: pytest + httpx `AsyncClient` (FastAPI standard)
- Types: unit tests for `DataService`, integration tests for endpoint
- Mock MinIO in unit tests; use test database for integration
