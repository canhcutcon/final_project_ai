# Story 2.4: Data Preview

## Status

Draft

## Story

**As a** user,
**I want** to preview the first 10 rows of my uploaded data,
**so that** I can verify the data was uploaded correctly.

## Acceptance Criteria

1: `GET /api/v1/upload/{id}/preview` trả về JSON với 10 dòng đầu
2: Response format: `{columns: [...], rows: [[...], ...], data_type: "tabular", stats: {column_name: {min, max, mean, null_count}}}`
3: Trả `404 Not Found` nếu dataset_id không tồn tại
4: Trả `403 Forbidden` nếu dataset không thuộc current user
5: Optional query param `?rows=N` để thay đổi số dòng preview (max 100)

## Tasks / Subtasks

- [ ] Task 1 — Define preview schemas (AC: 2)
  - [ ] Add to `{backend}/app/schemas/upload.py`:
  - [ ] Define `ColumnStats` schema: min (float | None), max (float | None), mean (float | None), null_count (int)
  - [ ] Define `PreviewResponse` schema: columns (list[str]), rows (list[list[Any]]), data_type (str | None), stats (dict[str, ColumnStats])

- [ ] Task 2 — Implement DataService.get_preview() (AC: 1, 2, 5)
  - [ ] Add `async def get_preview(dataset_id: UUID, rows: int = 10) -> PreviewResponse` to `{backend}/app/services/data_service.py`
  - [ ] Download CSV from MinIO via StorageService
  - [ ] Read first N rows with `pd.read_csv(nrows=rows)` for efficiency (avoid loading full file)
  - [ ] Compute per-column stats on full dataset (use `columns_info` from datasets table when available to avoid re-reading)
  - [ ] Build response: columns list, rows as list-of-lists, data_type from dataset record, stats dict

- [ ] Task 3 — Implement preview endpoint (AC: 1, 3, 4, 5)
  - [ ] Add `GET /api/v1/upload/{dataset_id}/preview` to `{backend}/app/api/v1/endpoints/upload.py`
  - [ ] Accept optional query param `rows: int = Query(default=10, ge=1, le=100)`
  - [ ] Lookup dataset by ID → raise 404 if not found
  - [ ] Check dataset.user_id == current_user.id → raise 403 if mismatch
  - [ ] Call `DataService.get_preview(dataset_id, rows)`
  - [ ] Return 200 with PreviewResponse
  - [ ] Register route in router

- [ ] Task 4 — Handle edge cases (AC: 1, 2)
  - [ ] If dataset has fewer rows than requested N, return all available rows
  - [ ] Handle non-numeric columns in stats: set min/max/mean to None, only provide null_count
  - [ ] Convert NaN/NaT values to None in JSON serialization (pandas default is NaN which is not valid JSON)

- [ ] Task 5 — Write tests (AC: 1, 2, 3, 4, 5)
  - [ ] Create `{backend}/tests/test_preview.py`
  - [ ] Test: preview returns default 10 rows with correct columns
  - [ ] Test: `?rows=5` returns exactly 5 rows
  - [ ] Test: `?rows=200` is rejected (> 100 max)
  - [ ] Test: non-existent dataset_id returns 404
  - [ ] Test: dataset owned by another user returns 403
  - [ ] Test: stats contain min/max/mean for numeric columns, None for categorical
  - [ ] Test: data_type field matches what was detected in Story 2.2

## Dev Notes

### Relevant Source Tree

- `{backend}/app/api/v1/endpoints/upload.py` — add preview GET endpoint
- `{backend}/app/services/data_service.py` — add `get_preview()` method
- `{backend}/app/schemas/upload.py` — add `ColumnStats`, `PreviewResponse`
- `{backend}/app/models/dataset.py` — read `columns_info`, `data_type`, `user_id`
- `{backend}/app/services/storage.py` — download CSV for preview
- `{backend}/app/api/deps.py` — `get_current_user` for auth check

### Patterns & Conventions

- Layered architecture: endpoint handles auth/validation, delegates to DataService
- HTTP status conventions: 200 success, 404 not found, 403 forbidden
- Query params validated by FastAPI `Query()` with ge/le constraints
- JSON serialization: ensure NaN → None conversion (pandas quirk)
- Read-only endpoint — no state mutation

### Tech Stack

- FastAPI (`Query` for optional params, `Path` for dataset_id)
- Pandas (`pd.read_csv(nrows=N)` for efficient partial read)
- SQLAlchemy 2.x (lookup dataset record)
- MinIO (download raw CSV)

### Notes from Story 2.1 & 2.2

- `columns_info` JSONB field already contains per-column null_count, unique_count — reuse for stats where possible to avoid re-reading full file
- `data_type` field populated by Story 2.2's auto-detection
- `Dataset` model has `user_id` FK for ownership check

### Testing

- Location: `{backend}/tests/`
- Framework: pytest + httpx `AsyncClient`
- Types: integration tests for endpoint (auth, 404, 403), unit tests for preview logic
- Need test fixtures: uploaded dataset with known content to verify row/column output
