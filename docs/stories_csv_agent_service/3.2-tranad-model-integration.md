# Story 3.2: TranAD Model Integration

## Status

Draft

## Story

**As a** data analyst,
**I want** to run TranAD model for transformer-based anomaly detection,
**so that** complex temporal patterns are captured.

## Acceptance Criteria

1: `{ai_services_detection}/src/models/tranad.py` implements `TranAD` class inheriting `BaseAnomalyModel`
2: `load_model(weights_path)` loads pretrained transformer weights
3: `predict(data: np.ndarray) -> AnomalyScores` runs inference
4: Support multivariate time-series: multiple feature columns simultaneously
5: Attention weights extracted to explain feature importance per anomaly
6: Returns `AnomalyScores` with `feature_importance: dict[str, float]` per anomaly row
7: Window-based scoring with configurable window size

## Tasks / Subtasks

- [ ] Task 1: Implement TranAD model architecture (AC: 1, 4)
  - [ ] Create `{ai_services_detection}/src/models/tranad.py`
  - [ ] Implement `TranAD` class inheriting `BaseAnomalyModel`
  - [ ] Implement transformer encoder with self-attention layers
  - [ ] Implement 2-phase adversarial decoder (Focus Score & reconstruction)
  - [ ] Support multivariate input: multiple feature columns handled simultaneously

- [ ] Task 2: Implement model loading (AC: 2)
  - [ ] Implement `load_model(weights_path: str)` for transformer state dict
  - [ ] Validate architecture config matches loaded weights (d_model, n_heads, n_layers)
  - [ ] Support loading from `.pth` file with config metadata

- [ ] Task 3: Implement attention-based feature importance (AC: 5, 6)
  - [ ] Hook into transformer attention layers to extract attention weights during forward pass
  - [ ] Aggregate attention weights across heads and layers per feature
  - [ ] Build `feature_importance: dict[str, float]` mapping feature names to importance scores
  - [ ] Attach feature importance to each anomaly row in results

- [ ] Task 4: Implement window-based predict method (AC: 3, 7)
  - [ ] Implement `predict(data: np.ndarray) -> AnomalyScores`
  - [ ] Apply window-based segmentation with configurable `window_size`
  - [ ] Calculate anomaly scores using reconstruction error + adversarial loss
  - [ ] Reuse sliding window from `{ai_services_detection}/src/data/transforms.py` (Story 3.1)

- [ ] Task 5: Extend AnomalyScores for feature importance (AC: 6)
  - [ ] Add optional `feature_importances: list[dict[str, float]]` field to `AnomalyScores` dataclass
  - [ ] Ensure backward compatibility — field defaults to `None` for models that don't provide it

- [ ] Task 6: Write tests (AC: 1-7)
  - [ ] Unit test: `TranAD` inherits `BaseAnomalyModel` correctly
  - [ ] Unit test: `load_model()` with valid transformer weights
  - [ ] Unit test: `predict()` with multivariate synthetic data
  - [ ] Unit test: attention extraction returns valid feature importance dict
  - [ ] Unit test: window-based scoring with different window sizes
  - [ ] Integration test: end-to-end inference with multi-feature time-series
