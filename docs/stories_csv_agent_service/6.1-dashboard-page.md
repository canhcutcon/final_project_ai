# Story 6.1: Dashboard Page

## Status

Draft

## Story

**As a** user,
**I want** a dashboard showing overview statistics and recent analyses,
**so that** I have a quick glance at my data activity.

## Acceptance Criteria

1: Route: `/dashboard` (protected — redirect to `/login` if not authenticated)
2: Stats cards: total datasets uploaded, total analyses run, average anomaly ratio
3: Recent analyses list: 5 most recent with columns: dataset name, model used, anomaly count, status, date
4: Click on analysis → navigate to `/analysis/{id}`
5: Donut chart: data type distribution (tabular vs timeseries vs mixed)
6: Bar chart: anomaly counts over time (last 7 days)
7: Responsive layout: 2-column grid on desktop, stacked on mobile
8: Dark mode styling with accent colors

## Dev Notes

### Relevant Source Tree

- `{frontend}/src/app/dashboard/page.tsx` — Dashboard page (new)
- `{frontend}/src/components/analysis/` — Analysis-related components (new)
- `{frontend}/src/hooks/useAnalysis.ts` — Data fetching hook (new)
- `{frontend}/src/lib/api.ts` — Axios/Fetch API client
- `{frontend}/src/styles/globals.css` — Global styles / dark mode

### Patterns & Conventions

- **Component naming**: PascalCase, colocated with feature directory
- **Feature-based organization**: components in `components/analysis/`
- **Custom hooks**: encapsulate API calls and state in `hooks/useAnalysis.ts`
- **State management**: React hooks (`useState`, `useEffect`) + custom hooks
- **Async state**: loading/error/data pattern in hooks
- **Pages**: Next.js App Router — `app/dashboard/page.tsx`

### Tech Stack

- **Next.js 14** — App Router, file-system routing
- **React 18** — components, hooks
- **TypeScript 5.x** — type safety
- **Recharts** — PieChart (donut), BarChart
- **Tailwind CSS** (inferred) — utility-first styling, responsive breakpoints, dark mode
- **Axios/Fetch** — API client in `lib/api.ts`

### Testing

- Test location: `{frontend}/src/__tests__/` or colocated `*.spec.tsx`
- Framework: Jest + React Testing Library (standard Next.js setup)
- Mock API responses in tests
- Test types: unit tests for components, hook behavior tests
