# Story 3.4: Model Selection & Inference API

## Status

Draft

## Story

**As a** user,
**I want** the system to automatically select the best model based on data type,
**so that** I get optimal detection results without manual configuration.

## Acceptance Criteria

1: `AIService.select_model(data_type)` mapping: `timeseries` → BiLSTM/TranAD (configurable), `tabular` → AnoGAN, `mixed` → ensemble
2: `POST /api/v1/analysis/detect` body: `{dataset_id, model_override?: string, config?: object}`
3: Endpoint flow: load data → select model → preprocess (if not done) → inference → save results
4: Results saved to `analysis_results` table with: `model_used`, `total_anomalies`, `anomaly_ratio`, `scores` (JSON), `metrics`, `duration_seconds`
5: `GET /api/v1/analysis/{id}/results` returns full results including per-row details: `[{row_idx, score, is_anomaly, contributing_features}]`
6: Metrics calculated: precision, recall, f1_score, auc_roc (if ground truth labels available)
7: `model_override` allows user to choose a specific model instead of auto-select

## Dev Notes

### Relevant Source Tree

- `{backend}/app/services/ai_service.py` — Core service for model selection and orchestration
- `{backend}/app/api/v1/endpoints/analysis.py` — REST endpoints for detect and results
- `{backend}/app/schemas/analysis.py` — Pydantic request/response DTOs
- `{backend}/app/models/analysis_result.py` — SQLAlchemy ORM model for `analysis_results` table
- `{ai_services_detection}/src/models/model_registry.py` — Model factory/registry
- `{ai_services_detection}/src/evaluation/metrics.py` — Evaluation metrics

### Patterns & Conventions

- **Layered architecture**: Endpoint (controller) → Service → Repository/Model
- **Dependency injection**: FastAPI `Depends()` via `{backend}/app/api/deps.py`
- **Async**: All service methods as `async def`
- **API versioning**: `/api/v1/` prefix
- **HTTP status**: `202` for async pipeline submission, `200` for GET results, `404` for not found
- **Validation**: Pydantic schemas enforced by FastAPI automatically
- **Error handling**: Custom exceptions in `{backend}/app/core/exceptions.py`

### Tech Stack

- **FastAPI** — endpoint definition, automatic OpenAPI docs
- **Pydantic** — request/response validation
- **SQLAlchemy 2.x** — ORM models, JSONB columns for scores/metrics
- **PostgreSQL 15+** — JSONB support for flexible score storage
- **Celery 5.x** — async task execution for inference (optional, can be synchronous initially)
- **scikit-learn** — precision_score, recall_score, f1_score, roc_auc_score

### Notes from Previous Stories

- Stories 3.1-3.3 provide the three model implementations: `BiLSTMAutoencoder`, `TranAD`, `AnoGAN`
- All models implement `BaseAnomalyModel` with `load_model()` and `predict()` interface
- `AnomalyScores` dataclass is the unified return type from all models
- TranAD provides `feature_importances`; BiLSTM and AnoGAN may not — handle gracefully
- Epic 2 `DataService` provides uploaded & preprocessed data via `dataset_id`

### Testing

- Test location: `{backend}/tests/` and `{ai_services_detection}/tests/`
- Framework: pytest
- Test types: unit tests for service/schema logic, integration tests for API roundtrip
- Mock model inference in API integration tests (avoid loading real weights)
