# Story 5.2: Pipeline Status Tracking

## Status

Draft

## Story

**As a** user,
**I want** to check the status of my running pipeline,
**so that** I know which step is executing and when it will complete.

## Acceptance Criteria

1: `POST /api/v1/pipeline/run` body: `{dataset_id, config}` → starts pipeline, returns `{pipeline_id, status: "pending"}`
2: `GET /api/v1/pipeline/{id}/status` returns full status with per-step details (name, status, duration_seconds, started_at)
3: Status transitions: `pending` → `running` → `completed` / `failed`
4: When completed: `result_id` and `report_id` populated in response
5: Returns `404` if pipeline_id does not exist
6: Returns `403` if pipeline does not belong to current user
7: WebSocket endpoint `ws://host/ws/pipeline/{id}` pushes status updates realtime (optional)

### Patterns & Conventions

- **Layered architecture**: Endpoint → Service → Model
- **Dependency injection**: FastAPI `Depends()` for DB session and auth
- **Async**: All endpoint and service methods as `async def`
- **API versioning**: `/api/v1/` prefix
- **HTTP status**: `202` for pipeline start, `200` for status query, `403` forbidden, `404` not found
- **WebSocket**: FastAPI native WebSocket support with `@app.websocket()` decorator
- **Status polling**: `GET /status` is baseline; WebSocket is optional enhancement
- **Error handling**: Custom exceptions in `{backend}/app/core/exceptions.py`

### Tech Stack

- **FastAPI** — REST endpoints + WebSocket support
- **Pydantic** — request/response validation
- **SQLAlchemy 2.x** — `pipeline_runs` + `pipeline_steps` models
- **Celery 5.x** — task callbacks for step status updates
- **Redis 7** — broker (also usable as pub/sub for WebSocket notifications)
- **Python 3.11+** — async/await, type hints

### Notes from Previous Stories

- Story 5.1 creates `PipelineService`, `pipeline_runs` model, Celery tasks, and `PipelineConfig`
- Story 5.1 tasks need updating to track per-step progress (this story extends them)
- `pipeline_runs` already has `status`, `current_step`, `error_message` — this story adds `pipeline_steps` for granular tracking
- Auth middleware from Epic 1 (Story 1.4) provides current user for ownership checks

### Testing

- Test location: `{backend}/tests/`
- Framework: pytest + `httpx.AsyncClient` for FastAPI testing
- Mock `PipelineService` in endpoint tests
- Use `CELERY_ALWAYS_EAGER=True` for integration tests
- WebSocket testing: use `websockets` or `starlette.testclient.TestClient`
