# Story 6.2: CSV Upload Page

## Status

Draft

## Story

**As a** user,
**I want** to drag-and-drop CSV files and preview data before analysis,
**so that** I can verify the correct file is uploaded.

## Acceptance Criteria

1: Route: `/upload` (protected)
2: `CSVUploader` component: drag-drop zone with visual feedback (border highlight, icon change)
3: File validation client-side: only `.csv`, max 100MB, display error message
4: Upload progress bar (percentage)
5: After successful upload: display `CSVPreview` with data table showing first 10 rows
6: `DataTypeIndicator` badge: display detected type (tabular/timeseries/mixed) with icon
7: `ModelSelector` dropdown: Auto (recommended) | BiLSTM | TranAD | AnoGAN
8: Button "Run Analysis" → call `POST /api/v1/analysis/detect` → redirect to analysis page
9: Button "Run Full Pipeline" → call `POST /api/v1/pipeline/run` → redirect to pipeline page

### Patterns & Conventions

- **Feature-based organization**: all CSV components in `components/csv/`
- **Custom hooks**: `useUpload` encapsulates upload logic, progress, and state
- **State management**: local `useState` for upload state, model selection
- **Async state**: loading/error/data pattern (idle/uploading/success/error)
- **Component naming**: PascalCase (`CSVUploader`, `CSVPreview`)
- **Business logic in hooks**: not in components

### Tech Stack

- **Next.js 14** — App Router
- **React 18** — hooks, event handlers for drag-drop
- **TypeScript 5.x** — type safety
- **Axios** — `onUploadProgress` for progress tracking
- **Tailwind CSS** — styling, responsive, dark mode
- **HTML5 Drag and Drop API** — native browser drag-drop
