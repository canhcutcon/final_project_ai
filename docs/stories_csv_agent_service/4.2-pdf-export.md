# Story 4.2: PDF Export

## Status

Draft

## Story

**As a** user,
**I want** to download the analysis report as a PDF,
**so that** I can share it with stakeholders.

## Acceptance Criteria

1: `NLPService.export_pdf(report_id)` converts Markdown content to PDF
2: PDF engine: ReportLab or WeasyPrint (supports Unicode Vietnamese)
3: PDF layout: header (logo + title), footer (page numbers), margins, professional typography
4: Includes charts/tables if present in report (rendered from data)
5: PDF saved to MinIO bucket `reports`: `{user_id}/{report_id}/report.pdf`
6: `pdf_path` updated in `reports` table
7: `GET /api/v1/report/{id}/download` streams PDF to client with header `Content-Disposition: attachment`
8: Returns `404` if report has no PDF, `403` if not owner

### Patterns & Conventions

- **Layered architecture**: Endpoint → Service → PDF Renderer + Storage
- **Dependency injection**: FastAPI `Depends()` for auth (ownership check)
- **Async**: All service methods as `async def`
- **API versioning**: `/api/v1/` prefix
- **HTTP status**: `201` for PDF creation, `200` for streaming download, `403` forbidden, `404` not found
- **File streaming**: Use FastAPI `StreamingResponse` for PDF download
- **Error handling**: Custom exceptions in `{backend}/app/core/exceptions.py`

### Tech Stack

- **WeasyPrint** (or ReportLab) — Markdown/HTML to PDF conversion with Unicode support
- **matplotlib** — chart rendering to PNG images for embedding
- **MinIO** — S3-compatible object storage, bucket `reports`
- **FastAPI** — `StreamingResponse` for file download
- **SQLAlchemy 2.x** — update `pdf_path` in `reports` table
- **Python 3.11+** — async/await, type hints
