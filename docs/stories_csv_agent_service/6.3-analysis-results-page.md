# Story 6.3: Analysis Results Page

## Status

Draft

## Story

**As a** user,
**I want** to see anomaly detection results with visual heatmaps and highlighted rows,
**so that** I can quickly identify problematic data.

## Acceptance Criteria

1: Route: `/analysis/{id}` (protected, verify ownership)
2: Summary header: model used, total rows, anomaly count, anomaly ratio, duration
3: `ScoreHeatmap` (Recharts): heatmap visualization of anomaly scores across rows/features
4: `AnomalyTable`: data table with rows sorted by anomaly score (descending), anomaly rows highlighted in red/orange
5: `AnomalyChart`: bar chart of anomaly score distribution (histogram)
6: Filter controls: slider for score threshold, toggle show anomalies only
7: Sort: by score, by row index, by specific feature
8: Metrics panel: precision, recall, F1, AUC (if available)
9: Button "Generate Report" → navigate to report generation

### Patterns & Conventions

- **Feature-based organization**: all analysis components in `components/analysis/`
- **Custom hooks**: `useAnalysis` extended with detail-fetching and filter state
- **State management**: local state for filters/sort; data from API hook
- **Component naming**: PascalCase
- **Dynamic routing**: Next.js `[id]` parameter via `useParams()`

### Tech Stack

- **Next.js 14** — App Router, dynamic route `[id]`
- **React 18** — hooks, state management
- **TypeScript 5.x** — type safety
- **Recharts** — BarChart (histogram), custom heatmap
- **Tailwind CSS** — styling, conditional row highlighting, dark mode

### Notes from Previous Stories

- Story 6.1 establishes auth guard and dashboard components
- Epic 3 (Story 3.4) provides `GET /api/v1/analysis/{id}/results` with per-row details
- Epic 4 (Story 4.1) provides `POST /api/v1/report/generate`
- Per-row data format: `{row_idx, score, is_anomaly, contributing_features}`
- TranAD results include `feature_importances` — use for heatmap feature axis
