# Story 3.1: BiLSTM Autoencoder Integration

## Status

Draft

## Story

**As a** data analyst,
**I want** to run BiLSTM Autoencoder on time-series data,
**so that** temporal anomalies are detected accurately.

## Acceptance Criteria

1: `{ai_services_detection}/src/models/bilstm_autoencoder.py` implements `BiLSTMAutoencoder` class inheriting `BaseAnomalyModel`
2: `load_model(weights_path)` loads pretrained weights from `.pth` file
3: `predict(data: np.ndarray) -> AnomalyScores` runs inference and returns scores per row
4: Sliding window approach for time-series: `window_size` configurable (default: 50)
5: Anomaly threshold auto-calculated: `mean + 2*std` of reconstruction errors
6: Support input shape: `(batch_size, sequence_length, n_features)`
7: Model inference runs on CPU (GPU optional)

## Tasks / Subtasks

- [ ] Task 1: Define BaseAnomalyModel abstract class (AC: 1)
  - [ ] Create `{ai_services_detection}/src/models/base.py` with abstract methods: `load_model()`, `predict()`, `get_threshold()`
  - [ ] Define `AnomalyScores` dataclass: `scores: np.ndarray`, `threshold: float`, `is_anomaly: np.ndarray`, `metadata: dict`
  - [ ] Define `ModelConfig` dataclass for shared configuration (device, batch_size, etc.)

- [ ] Task 2: Implement BiLSTMAutoencoder model architecture (AC: 1, 6)
  - [ ] Create `{ai_services_detection}/src/models/bilstm_autoencoder.py`
  - [ ] Implement encoder: BiLSTM layers that compress input sequences to latent representation
  - [ ] Implement decoder: BiLSTM layers that reconstruct input from latent representation
  - [ ] Support input shape `(batch_size, sequence_length, n_features)` with dynamic `n_features`

- [ ] Task 3: Implement model loading (AC: 2, 7)
  - [ ] Implement `load_model(weights_path: str)` to load `.pth` state dict
  - [ ] Auto-detect device (CPU/GPU) and move model accordingly
  - [ ] Add validation for weights file existence and compatibility

- [ ] Task 4: Implement sliding window preprocessing (AC: 4)
  - [ ] Create sliding window function in `{ai_services_detection}/src/data/transforms.py`
  - [ ] Accept `window_size` parameter (default: 50)
  - [ ] Handle edge cases: data shorter than window_size, padding strategy

- [ ] Task 5: Implement predict method with threshold calculation (AC: 3, 5)
  - [ ] Implement `predict(data: np.ndarray) -> AnomalyScores`
  - [ ] Calculate reconstruction errors per row (MSE between input and reconstruction)
  - [ ] Compute automatic threshold: `mean + 2*std` of reconstruction errors
  - [ ] Return `AnomalyScores` with per-row scores and anomaly flags

- [ ] Task 6: Write tests (AC: 1-7)
  - [ ] Unit test: `BiLSTMAutoencoder` inherits `BaseAnomalyModel` correctly
  - [ ] Unit test: `load_model()` with valid and invalid paths
  - [ ] Unit test: `predict()` returns correct `AnomalyScores` shape
  - [ ] Unit test: sliding window with various `window_size` values
  - [ ] Unit test: threshold calculation logic
  - [ ] Integration test: end-to-end inference with synthetic time-series data

### Patterns & Conventions

- Python: `snake_case` for files/functions, `PascalCase` for classes
- Docstrings for all classes/methods (triple-quote)
- Import order: Standard library → Third-party (torch, numpy) → Local
- Async not required for ML model classes (CPU-bound computation)

### Tech Stack

- **PyTorch 2.x** — model architecture, `torch.nn.Module`, `state_dict` loading
- **NumPy** — input/output data format (`np.ndarray`)
- **Python 3.11+** — type hints, dataclasses
- **scikit-learn** — `StandardScaler` if normalization needed in transforms

### Testing

- Test location: `{ai_services_detection}/tests/`
- Framework: pytest (standard Python testing)
- Test types: unit tests for each method, integration test for full inference pipeline
- Use synthetic data (random tensors) for tests — no pretrained weights required in CI
