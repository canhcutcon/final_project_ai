# Story 1.4: JWT Authentication Setup

## Status

Draft

## Story

**As a** user,
**I want** to register and login with JWT authentication,
**so that** my data and analysis results are secured.

## Acceptance Criteria

1. SQLAlchemy model `User` với fields: id, email (unique), hashed_password, full_name, created_at
2. `POST /auth/register` tạo user mới, hash password với bcrypt, trả về user info (không password)
3. `POST /auth/login` verify credentials, trả về `{"access_token": "...", "token_type": "bearer"}`
4. JWT token chứa `user_id` và `exp`, signed với `JWT_SECRET`
5. Dependency `get_current_user()` extract và verify token từ `Authorization: Bearer` header
6. Protected endpoints trả `401 Unauthorized` khi không có token hoặc token hết hạn
7. Token expiry configurable qua env var `JWT_EXPIRY_MINUTES` (default: 60)

### Patterns & Conventions

- **Auth flow**: `POST /auth/login` → JWT in response body → Client sends `Authorization: Bearer <token>` header
- **Password hashing**: bcrypt — never store plain text passwords
- **JWT payload**: Minimal — only `sub` (user_id) and `exp` (expiry). No sensitive data in token.
- **Dependency injection**: `get_current_user = Depends()` — use on any protected endpoint
- **Error responses**: `401` for auth failure, `409` for duplicate, `422` for validation
- **OAuth2 scheme**: FastAPI's `OAuth2PasswordBearer` — auto-generates auth in Swagger UI

### Tech Stack

- **PyJWT**: 2.8+ — JWT encode/decode
- **bcrypt**: 4.1+ — password hashing (argon2 alternative acceptable)
- **Pydantic**: v2 — request/response validation, `EmailStr` for email validation
- **FastAPI Security**: `OAuth2PasswordBearer` — token extraction from header

- **Minimum coverage**:

  - Login success/failure (wrong password, non-existent user)
  - Protected endpoint with/without valid token
  - Token expiry behavior
