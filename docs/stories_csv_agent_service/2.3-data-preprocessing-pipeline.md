# Story 2.3: Data Preprocessing Pipeline

## Status

Draft

## Story

**As a** user,
**I want** my data to be automatically cleaned and preprocessed,
**so that** it's ready for anomaly detection.

## Acceptance Criteria

1: `DataService.preprocess(dataset_id)` thực hiện pipeline: missing values → encoding → scaling
2: Missing values: drop rows nếu > 50% null, else fill với median (numeric) hoặc mode (categorical)
3: Categorical encoding: LabelEncoder cho low-cardinality (< 10 unique), one-hot cho medium (10-50), drop cho high (> 50)
4: Numeric scaling: StandardScaler (z-score normalization)
5: `PreprocessResult` chứa: columns_processed, rows_before, rows_after, transformations_applied (list)
6: Dữ liệu đã xử lý lưu vào MinIO: `{user_id}/{dataset_id}/processed.csv`
7: Scaler và encoder objects lưu lại để có thể inverse transform

  - [ ] Define `PreprocessResult` schema: columns_processed (int), rows_before (int), rows_after (int), transformations_applied (list[TransformationRecord])
  - [ ] Define `PreprocessResponse` schema: dataset_id, preprocess_result, processed_file_path

- [ ] Task 2 — Implement missing value handler (AC: 1, 2)
  - [ ] Create `{backend}/app/services/preprocessing/missing_handler.py`
  - [ ] Implement `handle_missing(df: DataFrame) -> tuple[DataFrame, list[TransformationRecord]]`
  - [ ] Drop rows where > 50% of columns are null
  - [ ] Fill numeric columns with median
  - [ ] Fill categorical columns with mode
  - [ ] Track each transformation applied (which columns, what strategy, how many values affected)

- [ ] Task 3 — Implement categorical encoder (AC: 1, 3, 7)
  - [ ] Create `{backend}/app/services/preprocessing/encoder.py`
  - [ ] Implement `encode_categoricals(df: DataFrame) -> tuple[DataFrame, list[TransformationRecord], dict]`
  - [ ] Low-cardinality (< 10 unique): apply `LabelEncoder`, store mapping
  - [ ] Medium-cardinality (10-50 unique): apply one-hot encoding via `pd.get_dummies()`
  - [ ] High-cardinality (> 50 unique): drop column, log warning

- [ ] Task 4 — Implement numeric scaler (AC: 1, 4, 7)
  - [ ] Create `{backend}/app/services/preprocessing/scaler.py`
  - [ ] Implement `scale_numerics(df: DataFrame) -> tuple[DataFrame, list[TransformationRecord], StandardScaler]`
  - [ ] Apply `StandardScaler` (z-score) to all numeric columns
  - [ ] Return fitted scaler object for inverse transform

- [ ] Task 5 — Implement DataService.preprocess() orchestrator (AC: 1, 5, 6, 7)
  - [ ] Add `async def preprocess(dataset_id: UUID) -> PreprocessResult` to `{backend}/app/services/data_service.py`
  - [ ] Download raw CSV from MinIO
  - [ ] Run pipeline in order: missing_handler → encoder → scaler
  - [ ] Collect all TransformationRecords
  - [ ] Save processed CSV to MinIO: `{user_id}/{dataset_id}/processed.csv`
  - [ ] Save scaler + encoder objects to MinIO: `{user_id}/{dataset_id}/transformers.pkl` (pickle serialized dict)
  - [ ] Build and return PreprocessResult

  - [ ] Validate dataset_id exists and belongs to current user (404 / 403)
  - [ ] Call `DataService.preprocess(dataset_id)`
  - [ ] Return 200 with PreprocessResponse

### Patterns & Conventions

- Layered architecture: endpoint → `DataService.preprocess()` → preprocessing submodules
- Each preprocessing step is a pure function: `(DataFrame) -> (DataFrame, transformations, fitted_object)`
- Pipeline order matters: missing → encode → scale (scale must be last, after encoding)
- Transformation records provide audit trail of what was done to the data
- Pickle for serializing sklearn objects (scaler, encoders) — standard practice

### Tech Stack

- Pandas (DataFrame manipulation, `fillna`, `get_dummies`)
- scikit-learn (`StandardScaler`, `LabelEncoder`)
- pickle (serialize transformer objects)
- MinIO (store processed CSV + transformer objects)
