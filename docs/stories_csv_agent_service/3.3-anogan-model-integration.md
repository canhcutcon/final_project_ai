# Story 3.3: AnoGAN Model Integration

## Status

Draft

## Story

**As a** data analyst,
**I want** to run AnoGAN for GAN-based anomaly detection on tabular data,
**so that** non-temporal anomalies are found.

## Acceptance Criteria

1: `{ai_services_detection}/src/models/anogan.py` implements `AnoGAN` class inheriting `BaseAnomalyModel`
2: `load_model(weights_path)` loads pretrained Generator and Discriminator weights
3: `predict(data: np.ndarray) -> AnomalyScores` calculates anomaly score = reconstruction_loss + discrimination_loss
4: Supports tabular data (no temporal dependency)
5: GAN-based data balancing: if positive/negative ratio < 0.1, generate synthetic samples
6: Anomaly score normalized to [0, 1] range
7: Batch inference supports large datasets (> 100K rows)

## Tasks / Subtasks

- [ ] Task 1: Implement AnoGAN model architecture (AC: 1)
  - [ ] Create `{ai_services_detection}/src/models/anogan.py`
  - [ ] Implement `AnoGAN` class inheriting `BaseAnomalyModel`
  - [ ] Implement Generator network (latent space ‚Üí tabular features)
  - [ ] Implement Discriminator network (tabular features ‚Üí real/fake score)
  - [ ] Implement latent space optimization (find z that best reconstructs input)

- [ ] Task 2: Implement model loading for dual networks (AC: 2)
  - [ ] Implement `load_model(weights_path: str)` to load both Generator and Discriminator
  - [ ] Support single `.pth` file with both state dicts or separate files
  - [ ] Validate Generator/Discriminator architecture compatibility

- [ ] Task 3: Implement anomaly scoring (AC: 3, 6)
  - [ ] Calculate reconstruction loss: MSE between input and Generator reconstruction
  - [ ] Calculate discrimination loss: Discriminator confidence score
  - [ ] Combine scores: `anomaly_score = alpha * reconstruction_loss + (1 - alpha) * discrimination_loss`
  - [ ] Normalize final scores to [0, 1] range using min-max normalization

- [ ] Task 4: Implement GAN-based data balancing (AC: 5)
  - [ ] Detect class imbalance: check positive/negative ratio
  - [ ] If ratio < 0.1, generate synthetic minority samples using Generator
  - [ ] Integrate balancing step as optional preprocessing in predict pipeline
  - [ ] Log balancing statistics (original count, synthetic count, final ratio)

- [ ] Task 5: Implement batch inference for large datasets (AC: 4, 7)
  - [ ] Implement batched `predict()` with configurable `batch_size`
  - [ ] Process data in chunks to avoid OOM on datasets > 100K rows
  - [ ] Aggregate results across batches into single `AnomalyScores` output
  - [ ] Support tabular data without temporal windowing (flat rows)

- [ ] Task 6: Write tests (AC: 1-7)
  - [ ] Unit test: `AnoGAN` inherits `BaseAnomalyModel` correctly
  - [ ] Unit test: `load_model()` loads both Generator and Discriminator
  - [ ] Unit test: anomaly score = reconstruction_loss + discrimination_loss
  - [ ] Unit test: score normalization produces values in [0, 1]
  - [ ] Unit test: data balancing triggers when ratio < 0.1
  - [ ] Unit test: batch inference with > 1000 rows (simulated large dataset)
  - [ ] Integration test: end-to-end inference with synthetic tabular data

### Tech Stack

- **PyTorch 2.x** ‚Äî Generator/Discriminator as `torch.nn.Module`, latent space optimization

- **Python 3.11+** ‚Äî type hints, dataclasses

### Notes from Previous Stories

- `BaseAnomalyModel` and `AnomalyScores` defined in Story 3.1

- `AnomalyScores.feature_importances` field (Story 3.2) is optional; AnoGAN may not populate it
