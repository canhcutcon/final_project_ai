# Story 4.1: LLM Report Generation

## Status

Draft

## Story

**As a** user,
**I want** the system to generate a natural language report explaining the anomalies found,
**so that** I can understand the results without deep technical knowledge.

## Acceptance Criteria

1: `NLPService.generate_report(anomaly_result, language, style)` creates Markdown report
2: Supports `language`: `vi` (Vietnamese) and `en` (English)
3: Supports `style`: `summary` (1-2 pages) and `detailed` (5-10 pages)
4: Chain-of-Thought prompting: LLM analyzes step-by-step before concluding
5: Report structure: Executive Summary, Anomaly Details (top-N), Statistical Overview, Recommendations
6: `POST /api/v1/report/generate` body: `{analysis_id, language?, style?}`
7: Report content saved to `reports` table (field `content` â€” Markdown text)
8: LLM provider configurable: OpenAI API (default) or Local Llama endpoint
9: Fallback: if LLM API fails, generate template-based report (no AI)

### Patterns & Conventions

- **Layered architecture**: Endpoint â†’ Service â†’ LLM Provider
- **Dependency injection**: FastAPI `Depends()` via `{backend}/app/api/deps.py`
- **Async**: All service methods as `async def`
- **API versioning**: `/api/v1/` prefix
- **HTTP status**: `201` for successful report creation, `422` for validation errors
- **Error handling**: Catch LLM provider errors in service layer, fall back to template
- **Config via env vars**: `LLM_PROVIDER`, `OPENAI_API_KEY`, `LLAMA_ENDPOINT_URL`

### Tech Stack

- **OpenAI API** â€” GPT chat completions for report generation
- **Local LLM (Llama)** â€” alternative provider via HTTP endpoint
- **FastAPI + Pydantic** â€” endpoint and validation
- **SQLAlchemy 2.x** â€” `reports` table with Text column for Markdown content
- **PostgreSQL 15+** â€” primary database
- **Python 3.11+** â€” async/await, type hints

### Notes from Previous Stories

- Epic 3 `analysis_results` table provides input: `model_used`, `total_anomalies`, `anomaly_ratio`, `scores` (JSONB), `metrics`
- `AnomalyScores` with per-row details and optional `feature_importances` from TranAD
- `analysis_id` links report to its source detection run
