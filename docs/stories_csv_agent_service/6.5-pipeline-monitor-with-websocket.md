# Story 6.5: Pipeline Monitor with WebSocket

## Status

Draft

## Story

**As a** user,
**I want** to see real-time pipeline progress with step-by-step updates,
**so that** I can monitor long-running analyses.

## Acceptance Criteria

1: Route: `/pipeline/{id}` (protected, verify ownership)
2: `PipelineProgress`: visual step progress bar with 5 steps, active step animated
3: Each step displays: name, status (pending/running/completed/failed), duration
4: `PipelineConfig` panel: display chosen config (auto_fix, language, report_style)
5: `useWebSocket` hook: connect to `ws://host/ws/pipeline/{id}`, auto-reconnect on disconnect
6: Fallback: if WebSocket not available, polling `GET /status` every 3 seconds
7: When completed: display links to Analysis Results and Report
8: When failed: display error message with "Retry" button
9: Real-time log output panel (scrollable, auto-scroll to bottom)

### Patterns & Conventions

- **Feature-based organization**: all pipeline components in `components/pipeline/`
- **Custom hooks**: `useWebSocket` encapsulates connection, reconnect, fallback logic
- **State management**: WebSocket-driven state updates; local state for UI
- **Component naming**: PascalCase
- **Dynamic routing**: Next.js `[id]` parameter
- **Real-time updates**: WebSocket is primary; polling is fallback — hook abstracts both

### Tech Stack

- **Next.js 14** — App Router, dynamic route
- **React 18** — hooks, `useRef` for scroll management, `useEffect` for cleanup
- **TypeScript 5.x** — type safety
- **WebSocket API** — native browser WebSocket
- **Tailwind CSS** — styling, animations (pulse for active step), dark mode
- **Axios** — polling fallback API calls
